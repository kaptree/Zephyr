# 阶段一：项目脚手架与基础设施

## 目标
搭建 Go 项目基础框架，完成项目结构初始化、配置系统、数据库连接、日志系统及中间件基础设施，为后续业务模块开发提供坚实基础。

## 技术要求

### 技术栈
- **语言**: Go 1.22
- **Web框架**: Gin v1.9+
- **ORM**: Gorm v2（PostgreSQL驱动）
- **配置管理**: 自定义JSON配置加载器（读取config.json）
- **日志**: Zap + Lumberjack（结构化日志、日志轮转）
- **数据库**: PostgreSQL 15+（通过Gorm连接）
- **缓存**: Redis（go-redis客户端）

### 项目目录结构
```
Server-code/
├── config.json              # 系统配置（统一参数管理）
├── config.json.template     # 配置模板文件
├── go.mod                   # Go模块定义
├── go.sum                   # 依赖校验
├── main.go                  # 程序入口
├── cmd/
│   └── server/
│       └── main.go          # 服务启动引导
├── internal/
│   ├── config/              # 配置加载模块
│   │   └── config.go
│   ├── database/            # 数据库连接与迁移
│   │   ├── postgres.go
│   │   └── redis.go
│   ├── logger/              # 日志模块
│   │   └── logger.go
│   ├── middleware/          # 中间件
│   │   ├── cors.go
│   │   ├── recovery.go
│   │   └── request_id.go
│   ├── models/              # 数据模型（Gorm）
│   ├── handlers/            # HTTP处理器
│   ├── services/            # 业务逻辑层
│   ├── repository/          # 数据访问层
│   ├── router/              # 路由注册
│   │   └── router.go
│   └── utils/               # 工具函数
│       ├── response.go      # 统一响应格式
│       └── validator.go     # 参数校验
├── pkg/                     # 可复用的公共包
│   └── errors/              # 自定义错误类型
└── tasks/                   # 开发任务文档
```
