# 阶段七：安全加固与系统优化

## 目标
实施全面的安全防护措施和系统优化，包括接口限流、文件上传安全、富文本XSS防护、SQL注入防护、日志审计和性能优化。

## 技术要求

### 1. 接口限流
- **策略**: 令牌桶算法，按IP+用户双维度限流
- **规则**: 登录接口 5次/分钟，普通API 100次/分钟
- **实现**: Gin中间件，Redis存储计数器
- **响应**: 429 Too Many Requests + Retry-After头

### 2. 文件上传安全
- **类型限制**: 白名单校验文件扩展名和MIME类型（doc/docx/xls/xlsx/pdf/png/jpg）
- **大小限制**: 单文件最大10MB
- **存储**: 本地磁盘或MinIO对象存储
- **病毒扫描**: 可选集成ClamAV

### 3. XSS防护
- 富文本内容服务端清洗：白名单标签机制
- 输入内容HTML实体编码
- API响应头：Content-Security-Policy

### 4. SQL注入防护
- 全部使用Gorm参数化查询
- 禁止字符串拼接SQL
- 可选PostgreSQL RLS行级安全策略

### 5. 安全请求头
- Helmet风格安全头中间件
- X-Content-Type-Options: nosniff
- X-Frame-Options: DENY
- X-XSS-Protection: 1; mode=block

### 6. 日志审计
- 所有API请求日志（含请求体、响应码、耗时）
- 敏感操作审计日志（登录、权限变更、删除）
- 日志脱敏（密码、Token不记录）

### 7. 性能优化
- 数据库连接池调优
- Redis缓存热点数据（用户权限、标签列表）
- Gorm查询优化（预加载、索引）
- Gzip压缩中间件
