# 阶段五：协同与工作组

## 目标
实现多人协同编辑便签和工作组管理功能，支撑实时协同画布、指令下发和在线状态同步。

## 技术要求

### 1. 协同房间（collaboration_rooms）
- 以 note_id 为唯一键，一个便签一个协同房间
- canvas_data 用 JSONB 存储分栏内容映射
- 版本号机制：服务端维护version，客户端提交需带last_version，不一致则全量同步
- 单栏编辑锁：用户聚焦某栏时广播"XX正在编辑"

### 2. 工作组（work_groups）
- 关联便签（note_id），支持子分组（sub_group_name）
- 成员角色：leader / member
- 状态流转：active → completed → dissolved
- 创建便签时可选创建工作组

### 3. WebSocket 实时通信
- **连接端点**: `/ws`（Socket.io兼容）或 `/ws/:note_id`（原生WebSocket）
- **认证**: 连接时Query参数传递token，服务端校验绑定user_id
- **房间管理**: 以note_id为Room ID

### 4. WebSocket 事件定义
| 事件 | 方向 | 说明 |
|------|------|------|
| `room:join` | C→S | 加入协同房间 |
| `room:leave` | C→S | 离开房间 |
| `canvas:update` | C→S | 栏位内容变更（100ms节流广播） |
| `canvas:sync` | S→C | 广播给房间内其他成员 |
| `canvas:command` | S→C | 领导指令下发 |
| `note:reminded` | S→C | 盯办推送 |
| `note:archived` | S→C | 归档通知 |
| `note:assigned` | S→C | 新任务下发 |
| `presence:update` | S→C | 在线成员列表 |

## API接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/rooms/:note_id/canvas` | 获取画布数据 |
| POST | `/api/v1/rooms/:note_id/command` | 推送指令到所有栏位 |
| POST | `/api/v1/groups` | 创建工作组 |
| GET | `/api/v1/groups/:id/members` | 工作组成员列表 |
| PUT | `/api/v1/groups/:id/members/:user_id` | 更新成员角色 |
| GET | `/ws/:note_id` | WebSocket连接 |
