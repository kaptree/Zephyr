# 阶段四：标签与模板管理

## 目标
实现标签库和工作模板的完整管理功能，支撑便签的分类体系和工作模板化创建。

## 技术要求

### 1. 标签管理
- **作用域**: personal（个人可见）/ system（全局可见）
- **创建权限**: system标签需dept_admin以上角色
- **删除保护**: 已关联便签的标签禁止删除，提示转移
- **颜色管理**: 十六进制色值存储
- **分类**: 工作类型/优先级/自定义

### 2. 模板管理
- **模板类型**: default/data_analysis/special_project/emergency_canvas/collaborative_writing
- **字段定义**: JSONB存储，支持 text/date/select/multi-select/rich-text 类型
- **布局**: 画布分栏数 1/2/4/6/8
- **权限**: system模板仅管理员可管理

## API接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/tags` | 标签列表（按scope/category筛选） |
| POST | `/api/v1/tags` | 创建标签 |
| PUT | `/api/v1/tags/:id` | 更新标签 |
| DELETE | `/api/v1/tags/:id` | 删除标签 |
| GET | `/api/v1/templates` | 模板列表（按type筛选） |
| GET | `/api/v1/templates/:id` | 模板详情（含字段定义） |
| POST | `/api/v1/templates` | 创建模板 |
| PUT | `/api/v1/templates/:id` | 更新模板 |
| DELETE | `/api/v1/templates/:id` | 删除模板 |
