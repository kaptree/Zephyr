# 阶段六：盯办提醒、台账与文号系统

## 目标

实现任务盯办督办机制、工作台账自动记录和文号管理系统，完善系统的督考和管理能力。

## 技术要求

### 1. 盯办提醒（reminders）

- **自动盯办**: 定时任务扫描 `due_time < NOW() + 2h AND is_archived=false` 的任务
- **人工盯办**: 发起人/组长/管理员手动触发，即时生效
- **桌面强警示**: WebSocket推送 + Redis离线补偿队列
- **紧急盯办**: remind_type=urgent时同步发送通知
- **盯办记录**: 每次盯办记录 target_id、message、确认状态

### 2. 工作台账（ledger_entries）

- **自动记录**: 每次创建/编辑/完成/归档/盯办/指派/导出操作
- **记录内容**: 操作类型、操作人、IP、UA、变更详情（JSONB）
- **查询统计**: 按时间/部门/用户/操作类型多维筛选
- **图表数据**: 按部门、按时间段的统计聚合

### 3. 文号系统（serial_no）

- **格式**: `资警轻燕〔YYYY〕XXXX号`
- **生成**: 任务创建时自动分配
- **一致性**: 数据库序列保证原子递增
- **预获取**: `/api/v1/serial-numbers/next` 接口支持预览

### 4. 导出功能

- 任务导出为Word/Excel文档（异步任务）
- 文件下载：`/api/v1/files/:task_id/download`

## API接口

| 方法 | 路径                              | 说明                        |
| ---- | --------------------------------- | --------------------------- |
| POST | `/api/v1/notes/:id/remind`        | 盯办提醒                    |
| GET  | `/api/v1/serial-numbers/next`     | 获取下一文号预览            |
| GET  | `/api/v1/ledger`                  | 台账列表（分页+多维筛选）   |
| GET  | `/api/v1/ledger/stats`            | 台账统计（图表数据）        |
| POST | `/api/v1/notes/:id/export`        | 导出报告（异步返回task_id） |
| GET  | `/api/v1/files/:task_id/download` | 下载导出文件                |
