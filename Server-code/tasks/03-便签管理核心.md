# 阶段三：任务管理核心

## 目标

实现任务的完整生命周期管理，包括创建、查询、更新、完成归档、恢复、删除等功能，构建系统的核心业务能力。

## 技术要求

### 1. 数据模型（Gorm）

- **notes** 主表：完整字段映射，含title、content、content_delta(JSONB)、color_status枚举、source_type枚举、template_type枚举、各关联外键、时间戳、软删除
- **note_tags** 关联表：联合主键(note_id, tag_id)
- **note_assignees** 关联表：含role_in_note枚举（initiator/leader/member/viewer）、反馈内容、已读状态
- **note_attachments** 表：附件信息

### 2. 核心业务逻辑

- **创建任务**: 事务内完成6步操作（校验→插入notes→关联tags→关联assignees→可选创建groups/rooms→台账记录）
- **权限过滤**: 根据用户角色动态构建SQL WHERE条件，实现行级数据隔离
- **文号生成**: 格式 `资警轻燕〔YYYY〕XXXX号`，年度重置，数据库序列保证原子性
- **状态流转**: active → completed → archived，支持恢复

### 3. 分页与筛选

- 支持多维度过滤：status/source_type/tag_ids/owner_id/creator_id/department_id/color_status
- 全文检索：keyword参数模糊匹配标题+内容
- 时间范围：date_from/date_to
- 紧急标记：is_urgent（服务端计算due_time与当前时间差）
- 排序：支持多字段排序

### 4. 文号（serial_no）

- 创建时自动分配
- PostgreSQL序列保证原子递增
- 年度重置：`资警轻燕〔2026〕0001号` → `资警轻燕〔2027〕0001号`

## API接口

| 方法   | 路径                         | 说明                        |
| ------ | ---------------------------- | --------------------------- |
| GET    | `/api/v1/notes`              | 任务列表（分页+多条件筛选） |
| POST   | `/api/v1/notes`              | 创建任务                    |
| GET    | `/api/v1/notes/:id`          | 任务详情（含关联数据）      |
| PUT    | `/api/v1/notes/:id`          | 更新任务                    |
| POST   | `/api/v1/notes/:id/complete` | 完成归档                    |
| POST   | `/api/v1/notes/:id/remind`   | 盯办提醒                    |
| DELETE | `/api/v1/notes/:id`          | 删除（软删除）              |
| POST   | `/api/v1/notes/:id/restore`  | 从归档恢复                  |
