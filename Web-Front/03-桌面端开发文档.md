# 轻燕数智·轻燕 —— 桌面端（WinUI 3）开发文档

> 版本：v1.0  
> 定位：轻量化情指行一体化支撑解决方案 · 桌面智能便签客户端  
> 核心载体：以"电脑桌面便签"为唯一统一入口，承载被动接单、主动记工、即时协同全场景

---

## 一、项目概述

桌面端是系统的**核心入口与高频操作端**，采用 C# + WinUI 3 技术栈开发，运行于 Windows 10/11。其设计理念为"无感嵌入办公环境"——以桌面悬浮便签为载体，不改变员工原有电脑操作习惯，实现任务接收、工作记录、协同反馈、盯办预警的极致轻量化体验。

## 三、设计规范（与Web端统一）

### 3.1 视觉风格
- **主背景**：纯白 `#FFFFFF` 为主，便签卡片使用毛玻璃（Mica/Acrylic）效果叠加纯白底，营造"悬浮于桌面"的轻盈感
- **便签圆角**：16px（与Web端一致），阴影 `Offset="0,4" BlurRadius="16" Color="#14000000"`
- **极简边框**：便签无外框线，仅通过阴影与背景区分层级

### 3.2 便签色彩语义（与Web/WebSocket状态强绑定）
- **黄色待办**：`#FEF3C7` 背景，接收后前3分钟左上角有呼吸灯式闪烁动画（`RepeatBehavior="Forever"`，周期2s）
- **绿色完成**：`#DCFCE7` 背景，右上角绿色对勾徽章，完成后卡片渐隐消失（`DoubleAnimation` 透明度0→消失，400ms）
- **红色盯办**：`#FEE2E2` 背景，边框 `#DC2626`，附加脉冲阴影动画（`ScaleTransform` 1.0→1.02循环），任务栏图标同步变红闪烁

### 3.3 字体与排版
- 字体：Segoe UI Variable / Microsoft YaHei UI（中文回退）
- 标题：16px SemiBold
- 正文：13px Regular，行高 1.5
- 标签胶囊：11px，内边距 4px 8px

### 3.4 交互动效
- 新建便签：从右下角悬浮按钮展开，卡片以 `ScaleTransform` 0.8→1.0 + `Opacity` 0→1 弹出，带弹性缓动（`CubicEase EasingMode="EaseOut"`）
- 拖拽移动：便签可自由拖拽于桌面任意位置，释放时轻微吸附对齐（8px网格吸附）
- 归档消失：卡片缩小至中心点 + 透明度归零，随后从视觉树移除
- 盯办震动：红色便签接收时，附加 `TranslateTransform` X方向微震动（±3px，3次，100ms）

## 五、本地数据与同步策略

### 5.1 SQLite本地库设计

**`LocalNotes`（本地便签缓存）**
- `Id` string（服务端UUID，本地主键）
- `Title` string
- `Content` string（HTML或纯文本）
- `ColorStatus` string（yellow/green/red）
- `SourceType` string
- `TagsJson` string（JSON数组，本地缓存标签）
- `CreatorId`, `OwnerId`, `AssignerId` string
- `DepartmentId` string
- `GroupId` string
- `IsArchived` bool
- `DueTime` DateTime?
- `CompletedAt` DateTime?
- `RemindCount` int
- `LastSyncAt` DateTime（最后同步时间）
- `LocalCreatedAt` DateTime（本地创建时间）
- `IsDirty` bool（本地是否有未同步的变更）
- `ScreenX`, `ScreenY` double（桌面坐标）
- `Width`, `Height` double（卡片尺寸，用户可调整）
- `IsMinimized` bool（是否最小化到侧边栏）

**`LocalUsers`（人员缓存）**
- `Id`, `Name`, `DepartmentId`, `Role`, `AvatarUrl`
- `LastSyncAt`

**`LocalTags`（标签缓存）**
- `Id`, `Name`, `Color`, `Scope`, `Category`

**`SyncQueue`（离线操作队列）**
- `Id` int PK
- `Operation` string（Create/Update/Complete/Remind/Delete）
- `EntityType` string（Note/Tag/...）
- `EntityId` string
- `PayloadJson` string
- `CreatedAt` DateTime
- `RetryCount` int

### 5.2 同步引擎（SyncEngine）

**同步策略**：
1. **启动同步**：应用启动时，先读本地缓存展示便签（秒开），后台异步调 `GET /api/v1/notes?status=active&updated_after={last_sync}` 增量拉取
2. **定时同步**：每5分钟轮询一次，WebSocket连接时以推送为主、轮询为兜底
3. **即时推送**：WebSocket连接保持，接收服务端事件：
   - `note:assigned` → 本地插入新便签，桌面生成黄色卡片，播放通知音
   - `note:reminded` → 更新本地便签为红色，触发震动动画与Toast
   - `note:archived` → 本地标记归档，播放消失动画后从视觉树移除
   - `canvas:sync` → 更新协同画布内容（仅即时协同模式）
4. **离线队列**：网络断开时，用户操作（创建、编辑、完成）写入 `SyncQueue`，恢复后按序批量提交（`POST /api/v1/notes/batch`）
5. **冲突解决**：本地 `IsDirty=true` 且服务端返回更新时，比较 `updated_at`，服务端新则覆盖本地，本地新则提示用户选择

## 六、核心业务逻辑与接口调用

### 6.1 便签生命周期（桌面端视角）

```
接收任务/新建便签 → 桌面生成卡片（黄/白） → 编辑内容/贴标签 → （协同反馈） → 点击完成 → 本地归档动画 → 调用complete接口 → 服务端确认 → 彻底移除卡片
```

#### 6.1.1 被动接收任务（场景1）
- **触发**：WebSocket收到 `note:assigned` 事件，或轮询发现新 `assigned` 便签
- **桌面表现**：
  1. 屏幕右下角（或上次记忆位置）生成黄色便签卡片
  2. 标题栏显示"【待办】数据分析任务"，来源人姓名
  3. 前3分钟左上角呼吸灯闪烁，系统播放轻提示音（可关闭）
  4. 托盘图标变黄
- **用户操作**：
  - 点击卡片展开详情，内置 `RichEditBox` 编辑区
  - 边工作边记录，内容自动本地保存（防抖500ms写入SQLite）
  - 点击"完成"按钮 → 二次确认弹窗 → 调用 `POST /api/v1/notes/{id}/complete` → 本地播放归档动画 → 卡片消失
  - 若网络异常，按钮显示"稍后同步"，入队列待提交

#### 6.1.2 主动发起工作（场景2）
- **触发**：用户点击托盘"新建便签"或快捷键 `Ctrl+Alt+N`
- **流程**：
  1. 桌面生成空白便签（默认白色背景，非任务状态）
  2. 用户输入标题与内容，顶部 `TagSelector` 贴标签（如"协助查询"）
  3. 内容实时本地保存，无网络亦可使用
  4. 用户可随时点击"归档"（非完成，只是个人工作记录归档）或"完成"（正式提交）
  5. 调用 `POST /api/v1/notes` 或 `PUT /api/v1/notes/{id}` 同步到云端
  6. 同步成功后，Web端可按标签/时间检索该记录

#### 6.1.3 多人协同工作（场景3/4）
- **触发**：用户在便签上开启"协同模式"，选择人员后调用服务端创建协同便签
- **桌面表现**：
  - 发起人桌面：便签变为"协同中"状态，底部显示参与人头像列表
  - 参与人桌面：收到 `note:assigned` 推送，生成黄色便签
  - 各参与人编辑反馈内容，点击"提交反馈"，内容汇总至发起人
- **专项分组（场景4）**：
  - 发起人可分组，桌面端显示组名与组长标识
  - 组长可点击"盯办组员"，调用 `POST /api/v1/notes/{id}/remind`，被盯办人桌面便签变红

#### 6.1.4 即时协同（场景5）
- **触发**：选择"即时协同模板"，指定分栏与人员
- **桌面表现**：
  - 便签展开为宽幅画布（600px+），按选择分栏（4栏/6栏）
  - 每栏绑定用户，顶部显示部门+姓名
  - WebSocket连接 `room:join`，实时接收其他栏内容变更
  - 用户在自己栏位输入，本地 `RichEditBox` onChange 节流300ms发送 `canvas:update`
  - 领导指令以红色横幅形式插入画布顶部
  - 可点击"投屏"按钮，将当前画布URL发送至大屏端

### 6.2 权限控制（桌面端校验）

桌面端本地维护一份 `UserPermissions` 缓存（登录时从 `GET /api/v1/auth/me` 获取）：

| 权限位 | 说明 | 桌面端表现 |
|--------|------|-----------|
| `can_create_self` | 创建个人便签 | 始终允许 |
| `can_create_assigned` | 为他人指派任务 | 创建弹窗显示"指派给他人"选项卡 |
| `can_remind` | 盯办他人 | 协同便签上显示"盯办"按钮 |
| `can_view_dept_archive` | 查看部门归档 | 托盘菜单显示"打开部门台账"入口 |
| `can_admin` | 系统管理 | 托盘菜单显示"管理后台"入口 |

**本地操作预校验**：点击按钮前，先查本地权限缓存，无权限则提示"无权操作"，避免无效请求。权限变更时WebSocket推送更新，即时生效。

### 6.3 标签贴附逻辑
- 桌面端标签来自本地缓存 `LocalTags`，启动时同步服务端
- 用户输入标签时，自动补全（本地SQLite `LIKE` 查询）
- 新建标签时，若scope=personal，仅写入本地与同步个人标签；若需创建系统标签，提示"无权限"或自动转为personal
- 一个便签最多贴5个标签，超出提示
- 标签在卡片上以 `ItemsControl` 横向排列胶囊形态展示

### 6.4 盯办红色强警示机制
- **接收盯办**：WebSocket `note:reminded` → 更新本地便签 `ColorStatus=red`
- **UI表现**：
  - 卡片边框变红，启动脉冲阴影动画
  - 卡片 `Topmost=true` 强制置顶
  - 托盘图标变红闪烁
  - 系统Toast通知："【盯办】王队提醒您尽快完成数据分析任务"
  - 若5分钟内未点击卡片，播放一次提示音
- **解除盯办**：用户打开卡片并编辑内容/点击"知道了"，调用 `POST /api/v1/notes/{id}/acknowledge_remind`（如需要），卡片红色解除（但任务仍待完成）

## 七、接口调用规范

桌面端通过 Refit 定义强类型接口，BaseURL 配置于 `appsettings.json`。

### 7.1 认证相关

| 接口 | 方法 | Refit定义 | 说明 |
|------|------|----------|------|
| 登录 | POST | `[Post("/auth/login")] Task<ApiResponse<LoginResult>> Login(LoginRequest req)` | 成功后保存token与refresh_token到Windows凭据保险箱（Credential Locker） |
| 刷新令牌 | POST | `[Post("/auth/refresh")] Task<ApiResponse<TokenPair>> Refresh([Body] RefreshRequest req)` | Access Token过期前自动调用 |
| 获取当前用户 | GET | `[Get("/auth/me")] Task<ApiResponse<UserProfile>> GetCurrentUser()` | 启动时调用，缓存权限矩阵 |

### 7.2 便签管理

| 接口 | 方法 | Refit定义 | 桌面端调用时机 |
|------|------|----------|--------------|
| 获取便签列表 | GET | `[Get("/notes")] Task<ApiResponse<PagedList<Note>>> GetNotes([Query] NoteQueryRequest req)` | 启动同步、定时轮询、下拉刷新 |
| 创建便签 | POST | `[Post("/notes")] Task<ApiResponse<Note>> CreateNote([Body] CreateNoteRequest req)` | 用户新建并首次同步时 |
| 更新便签 | PUT | `[Put("/notes/{id}")] Task<ApiResponse<Note>> UpdateNote(string id, [Body] UpdateNoteRequest req)` | 内容编辑防抖同步（5秒无操作后调用） |
| 完成便签 | POST | `[Post("/notes/{id}/complete")] Task<ApiResponse<Note>> CompleteNote(string id, [Body] CompleteRequest req)` | 用户点击"完成"按钮 |
| 盯办便签 | POST | `[Post("/notes/{id}/remind")] Task<ApiResponse<Note>> RemindNote(string id, [Body] RemindRequest req)` | 组长/发起人点击"盯办" |
| 删除便签 | DELETE | `[Delete("/notes/{id}")] Task<ApiResponse<object>> DeleteNote(string id, [Query] bool soft=true)` | 用户删除或归档个人便签 |
| 恢复便签 | POST | `[Post("/notes/{id}/restore")] Task<ApiResponse<Note>> RestoreNote(string id)` | 从归档恢复（需打开Web端或本地缓存操作） |
| 批量同步 | POST | `[Post("/notes/batch")] Task<ApiResponse<BatchResult>> BatchSync([Body] List<SyncOperation> ops)` | 离线恢复后批量提交队列 |

### 7.3 标签与组织

| 接口 | 方法 | Refit定义 | 桌面端调用时机 |
|------|------|----------|--------------|
| 获取标签 | GET | `[Get("/tags")] Task<ApiResponse<List<Tag>>> GetTags([Query] string scope)` | 启动同步，本地SQLite缓存 |
| 创建标签 | POST | `[Post("/tags")] Task<ApiResponse<Tag>> CreateTag([Body] CreateTagRequest req)` | 用户新建个人标签 |
| 获取部门树 | GET | `[Get("/departments")] Task<ApiResponse<List<Department>>> GetDepartments()` | 启动同步，人员选择器使用 |
| 获取可见人员 | GET | `[Get("/users/visible")] Task<ApiResponse<List<User>>> GetVisibleUsers()` | 协同选人时预加载 |

### 7.4 协同与模板

| 接口 | 方法 | Refit定义 | 桌面端调用时机 |
|------|------|----------|--------------|
| 获取模板 | GET | `[Get("/templates")] Task<ApiResponse<List<Template>>> GetTemplates([Query] string type)` | 新建便签时弹窗加载 |
| 加入协同房间 | WS | `Socket.Emit("room:join", new { note_id })` | 打开协同便签时 |
| 推送画布变更 | WS | `Socket.Emit("canvas:update", payload)` | 协同编辑节流发送 |

### 7.5 导出与文号

| 接口 | 方法 | Refit定义 | 桌面端调用时机 |
|------|------|----------|--------------|

## 八、离线模式与弱网适配

### 8.1 离线检测
- 网络状态监听 `NetworkInformation.NetworkStatusChanged`
- 离线时：托盘图标变灰，显示"离线模式"Tooltip，所有写操作入 `SyncQueue`

### 8.2 本地优先策略
- 用户输入内容**始终先写SQLite**，后台异步同步服务端
- 即使离线，用户可无限新建、编辑、完成便签，体验无中断
- 离线完成的便签，在本地标记 `IsArchived=true`，网络恢复后批量提交

### 8.3 数据一致性保障
- 每次同步携带本地 `LastSyncAt`，服务端返回增量变更
- 同步冲突时（同一便签本地与远端均变更），桌面端弹窗提示：
  - "该便签在Web端已被修改，请选择：保留本地 / 使用云端 / 合并查看"
- 定时全量校验：每周一次（或手动触发）全量拉取比对，修复本地与服务端差异

## 九、系统集成与性能优化

### 9.1 开机启动
- 通过 `Windows.ApplicationModel.StartupTask` 申请开机启动权限
- 开机后静默启动，仅托盘图标，不主动弹出便签（除非有待办任务）

### 9.2 内存与性能
- 便签卡片采用 `VirtualizingPanel` 思想，超出屏幕的便签暂停渲染
- 富文本编辑器限制图片大小（自动压缩至宽度800px）
- 本地SQLite定期VACUUM，清理已归档超过1年的便签（保留服务端）
- 图片附件按需加载，不自动下载，点击预览时拉取

### 9.3 快捷键
| 快捷键 | 功能 |
|--------|------|
| `Ctrl + Alt + N` | 新建便签 |
| `Ctrl + Alt + H` | 显示/隐藏所有便签 |
| `Ctrl + Enter` | 保存并关闭当前便签编辑 |
| `Ctrl + Shift + A` | 快速归档当前焦点便签 |
| `Esc` | 关闭弹窗/退出编辑模式 |

### 9.4 多用户与数据隔离
- 桌面端绑定当前Windows登录用户，配置与SQLite存储于 `%LocalAppData%\轻燕数智轻燕\{user_sid}\`
- 切换Windows用户时，应用数据完全隔离
