### 6.2 便签墙核心功能

#### 6.2.1 列表加载
- 页面加载时调用 `useNoteStore.fetchNotes({ status: 'active' })`
- 首次加载：显示骨架屏（6个卡片占位，脉冲动画）
- 后续加载：增量更新，保留当前列表
- 错误重试：加载失败显示"重新加载"按钮

#### 6.2.2 筛选与排序
- 状态筛选：全部 / 待办 / 盯办 / 已完成
- 排序方式：创建时间、更新时间、优先级
- 筛选状态同步到 URL query 参数

#### 6.2.3 空态
- 纯白底中央插画（便签图标 SVG）
- 文字："点击右下角 '+' 新建便签"
- 无冗余元素，极简风格

#### 6.2.4 拖拽排序
- 便签卡片支持拖拽重排（HTML5 Drag & Drop 或第三方库）
- 排序结果本地记忆（localStorage）
- 可选同步服务端

#### 6.2.5 虚拟滚动（>100张便签时启用）
- 使用 `@tanstack/vue-virtual` 或自行实现
- 仅渲染可视区域 + 上下缓冲区内的卡片

### 6.3 新建便签模态框

#### 触发
- 点击右下角悬浮按钮（圆形，56×56px，`#3B82F6`，带阴影 `0 4px 12px rgba(59,130,246,0.4)`）

#### 模态框内容
```
┌─────────────────────────────────────┐
│  新建便签                    [× 关闭]│
├─────────────────────────────────────┤
│  [标题输入框，自动聚焦]              │
│                                      │
│  [富文本内容编辑区]                   │
│                                      │
│  ┌ 标签选择 ─────────────────────┐  │
│  │ [标签1 ×] [标签2 ×] [+ 添加]  │  │
│  └────────────────────────────────┘  │
│                                      │
│  类型：○仅自己  ○指派他人  ○开启协同  │
│  [人员选择器（指派/协同时显示）]     │
│                                      │
│  截止时间：[日期选择器]              │
│                                      │
├─────────────────────────────────────┤
│              [取消]    [创建便签]     │
└─────────────────────────────────────┘
```

#### 创建流程
1. 用户填写信息，点击"创建"
2. 调用 `useNoteStore.createNote(payload)`
3. 成功后：模态框关闭
4. 新卡片以弹簧动画插入列表首位
5. 失败：表单内显示错误信息，不关闭模态框
6. 如果 owner_id ≠ 自己：推送通知桌面端

### 6.4 便签详情侧滑面板

#### 触发
- 单击便签卡片

#### 面板设计
- 右侧滑出（宽度 480px）
- 半透明遮罩覆盖左侧
- 面板内显示完整便签信息：
  - 标题（可编辑）
  - 富文本内容（可编辑）
  - 标签（可编辑）
  - 创建人、创建时间
  - 指派人列表
  - 历史操作记录

#### 操作按钮
- 编辑保存
- 完成归档
- 盯办提醒
- 删除/归档
- 恢复（仅已完成/已归档便签）

#### 面板关闭
- 点击遮罩
- 点击关闭按钮
- 按 Escape 键

### 6.5 归档动画

- 点击"完成"后，便签卡片播放消失动画
- 动画序列：`opacity: 1→0`, `scale: 1→0.95`, `translateY: 0→20px`
- 持续时间：400ms，`cubic-bezier(0.4, 0, 0.2, 1)`
- 动画完成后从 DOM 移除

### 6.6 NoteStore 完善

实现 `useNoteStore` 完整业务逻辑：
- `fetchNotes(filters)` — 调用 `GET /api/v1/notes`，支持分页
- `createNote(payload)` — 调用 `POST /api/v1/notes`，成功后 prepend
- `updateNote(id, payload)` — 乐观更新模式（先改本地，失败回滚）
- `completeNote(id, feedback?)` — 调用 `POST /api/v1/notes/{id}/complete`
- `remindNote(id, message?)` — 调用 `POST /api/v1/notes/{id}/remind`
- `archiveNote(id)` — 调用 `DELETE /api/v1/notes/{id}`
- `restoreNote(id)` — 调用 `POST /api/v1/notes/{id}/restore`
